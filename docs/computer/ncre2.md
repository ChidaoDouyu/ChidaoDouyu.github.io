## 一、计算机系统

### 1. 计算机系统结构掌握要点
1. 冯·诺依曼体系结构（五大核心部件：运算器、控制器、存储器、输入设备、输出设备；核心思想：存储程序与程序控制）
2. 计算机系统的层次结构（硬件层 → 系统软件层 → 应用软件层）
3. 计算机系统的分类（按规模：巨型机/大型机/小型机/微型机；按用途：通用机/专用机）
4. 计算机主要技术指标（字长、主频、存储容量（内存/外存）、运算速度（MIPS）、I/O带宽）

### 2. 计算机硬件系统结构掌握要点
1. CPU：功能（指令控制、操作控制、时间控制、数据加工）；组成（运算器、控制器、寄存器组）
2. 存储器分层体系：高速缓存（Cache）、内存（RAM/ROM）、外存（硬盘/U盘/光盘）的层级关系与特性
3. 总线：数据总线、地址总线、控制总线的功能与区别
4. 外部设备：输入设备（键盘/鼠标/扫描仪等）、输出设备（显示器/打印机等）；I/O接口的作用
5. 数据的二进制表示与编码：进制转换（二进制↔十进制/八进制/十六进制）；整数编码（原码/反码/补码/偏移码）；字符编码（ASCII码/Unicode/UTF-8）

### 3. 操作系统基本组成掌握要点
1. 操作系统核心特性（并发性、共享性、虚拟性、异步性）
2. 进程管理：进程概念、进程状态转换、进程调度、线程与进程的区别、并发与并行
3. 内存管理：分区管理、分页/分段管理、虚拟内存技术、页面置换算法
4. 目录和文件系统：文件属性、多级目录（树型结构）、文件操作、文件权限
5. I/O设备管理：设备分类、I/O控制方式、SPOOLING技术、设备分配策略

### 4. 计算机网络基础掌握要点
1. 网络的分类（按地理范围：局域网、广域网、互联网）
2. OSI/RM与TCP/IP协议模型（层级、核心协议）
3. 网络安全基础（防火墙、加密技术、入侵检测）
4. 互联网应用（WWW、电子邮件、FTP等）

## 二、基本数据结构与算法

### 1. 算法的基本概念；算法复杂度的概念和意义（时间复杂度与空间复杂度）。
1. 算法的基本特征（有穷性、确定性、可行性、输入、输出）
2. 算法的基本设计方法（列举法、归纳法、递推法、递归法、减半递推、回溯法）
3. 时间复杂度：定义、常见量级（O(1)/O(n)/O(n²)/O(log₂n)）及计算
4. 空间复杂度：定义、辅助空间计算、时间/空间复杂度的权衡

### 2. 数据结构的定义；数据的逻辑结构与存储结构；数据结构的图形表示；线性结构与非线性结构的概念。
1. 数据结构组成（数据元素、数据项、数据关系）
2. 逻辑结构（集合结构、线性结构、树形结构、图形结构）
3. 存储结构（顺序存储、链式存储、索引存储、散列存储的特点）
4. 数据结构图形表示（顶点、边的表示方法）
5. 线性结构（唯一根节点，每个节点最多1个前驱/后继）与非线性结构（树形/图形结构）的判定

### 3. 线性表的定义；线性表的顺序存储结构及其插入与删除运算。
1. 线性表的逻辑特征（有序、有限序列）
2. 顺序存储结构：存储特点、地址计算、优缺点
3. 顺序表插入：位置合法性判断、元素后移、时间复杂度O(n)
4. 顺序表删除：元素前移、时间复杂度O(n)、空/满表处理
5. 顺序表查找（按值/按位查找）

### 4. 栈和队列的定义；栈和队列的顺序存储结构及其基本运算。
1. 栈：定义（后进先出LIFO）、顺序栈（栈顶指针、空/满判定）、基本运算（入栈/出栈/读栈顶）
2. 队列：定义（先进先出FIFO）、循环队列（队头/队尾指针、空/满判定）、基本运算（入队/出队/读队头）
3. 栈和队列的应用场景（表达式求值、递归、缓冲处理）

### 5. 线性单链表、双向链表与循环链表的结构及其基本运算。
1. 链表存储特点（非连续存储、节点组成：数据域+指针域）
2. 单链表：头指针/头节点、空链表判定、查找/插入/删除运算
3. 双向链表：节点组成（数据域+前驱/后继指针）、插入/删除运算
4. 循环链表：单/双循环链表结构特点（尾节点指向头节点）
5. 链表与顺序表的对比

### 6. 树的基本概念；二叉树的定义及其存储结构；二叉树的前序、中序和后序遍历。
1. 树的基本概念（节点、度、叶子节点、深度、层次、双亲/孩子/兄弟节点）
2. 二叉树定义（每个节点最多2个子树，左右子树有序）及核心性质（第k层最多2^(k-1)个节点等）
3. 二叉树存储结构（顺序存储、二叉链表）
4. 二叉树遍历：前序（根-左-右）、中序（左-根-右）、后序（左-右-根）的遍历规则；已知两种遍历还原二叉树

### 7. 顺序查找与二分法查找算法；基本排序算法（交换类排序，选择类排序，插入类排序）。
1. 顺序查找：适用场景、算法步骤、时间复杂度O(n)
2. 二分法查找：适用条件（有序顺序表）、算法步骤、时间复杂度O(log₂n)
3. 交换类排序：冒泡排序、快速排序（步骤、时间复杂度、稳定性）
4. 选择类排序：简单选择排序、堆排序（步骤、时间复杂度、稳定性）
5. 插入类排序：直接插入排序、希尔排序（步骤、时间复杂度、稳定性）
6. 排序算法稳定性判定（相同关键字元素相对位置是否不变）

## 三、程序设计基础

### 1. 程序设计方法与风格。
1. 良好程序设计风格要求（源程序文档化、数据说明规范化、语句简单化、输入输出人性化）
2. 程序设计方法发展（结构化方法、面向对象方法）
3. 可读性优先的设计原则

### 2. 结构化程序设计。
1. 核心思想（自顶向下、逐步求精、模块化、限制goto语句）
2. 基本结构（顺序结构、选择结构（分支）、循环结构（当型/直到型））
3. 结构化程序设计的优缺点

### 3. 面向对象的程序设计方法，对象，方法，属性及继承与多态性。
1. 面向对象基本概念（对象、类、实例、消息）
2. 对象特征（标识唯一性、分类性、多态性、封装性、模块独立性）
3. 方法（对象的行为，类的成员函数）；属性（对象的状态，类的成员变量）
4. 继承：定义、单/多继承、父类/子类、代码复用优势
5. 多态性：编译时/运行时多态、重载/重写实现方式
6. 封装性：隐藏内部细节，仅通过接口交互

## 四、软件工程基础

### 1. 软件工程基本概念，软件生命周期概念，软件工具与软件开发环境。
1. 软件工程定义（工程化思想开发/维护软件）及核心思想
2. 软件生命周期（可行性研究→需求分析→设计→编码→测试→运行维护）
3. 软件工具（需求分析/设计/编码/测试工具分类）
4. 软件开发环境（IDE：集成式开发环境，如PyCharm/Visual Studio）
5. 软件工程基本原则（抽象、信息隐藏、模块化、高内聚低耦合）

### 2. 结构化分析方法，数据流图，数据字典，软件需求规格说明书。
1. 结构化分析（SA）核心（自顶向下、逐步分解、面向数据流）
2. 数据流图（DFD）：组成（外部实体、处理、数据流、数据存储）、绘制规则、分层DFD
3. 数据字典（DD）：DFD元素定义集合（数据流/存储/处理/外部实体说明）
4. 软件需求规格说明书（SRS）：作用、核心内容、编写要求（完整性/一致性/可验证性）

### 3. 结构化设计方法，总体设计与详细设计。
1. 结构化设计（SD）核心（将需求转化为软件结构，面向模块）
2. 总体设计（概要设计）：模块划分、模块接口、数据结构设计；模块独立性（内聚性/耦合性、高内聚低耦合）
3. 详细设计：模块内部算法/数据结构设计；设计工具（程序流程图、N-S图、PAD图、PDL）

### 4. 软件测试的方法，白盒测试与黑盒测试，测试用例设计，软件测试的实施，单元测试、集成测试和系统测试。
1. 软件测试目的（发现错误，非证明无错）及原则（尽早测试、独立测试、回归测试）
2. 白盒测试：关注内部逻辑；方法（逻辑覆盖、基本路径测试）
3. 黑盒测试：关注功能需求；方法（等价类划分、边界值分析、错误推测法）
4. 测试用例设计（输入数据、预期输出、测试步骤）
5. 测试实施步骤：单元测试（模块级，白盒为主）→集成测试（接口级，增量式/非增量式）→系统测试（系统级，黑盒为主）→验收测试

### 5. 程序的调试，静态调试与动态调试。
1. 调试与测试的区别（调试：定位/修复错误；测试：发现错误）
2. 静态调试：不执行程序，阅读代码/走查/静态分析工具
3. 动态调试：执行程序，断点/单步执行/日志输出定位错误
4. 调试方法（强行排错法、回溯法、原因排除法）
5. 调试步骤（发现→定位→修复→验证）

## 五、数据库设计基础

### 1. 数据库的基本概念：数据库，数据库管理系统，数据库系统。
1. 数据管理技术发展（人工管理→文件系统→数据库系统）
2. 数据库（DB）：长期存储、有组织、可共享的数据集
3. 数据库管理系统（DBMS）：核心功能（数据定义/操纵/运行管理/维护）
4. 数据库系统（DBS）：组成（DB、DBMS、硬件、软件、用户（最终用户/DBA/程序员））
5. 数据库系统特点（数据共享性、独立性（物理/逻辑）、结构化、统一数据控制）

### 2. 数据模型，实体联系模型及E - R图，从E - R图导出关系数据模型。
1. 数据模型组成（数据结构、数据操作、数据约束）；分类（概念/逻辑/物理模型）
2. E-R模型：实体（矩形）、属性（椭圆形）、联系（菱形，1:1/1:n/m:n）；E-R图绘制
3. 关系数据模型：层次/网状/关系模型分类；E-R图转关系模型（实体→表、属性→字段、联系→表/字段）

### 3. 关系代数运算，包括集合运算及选择、投影、连接运算，数据库规范化理论。
1. 关系模型基本概念（关系/元组/属性/主键/外键/关系模式）
2. 关系代数运算：集合运算（并/交/差/笛卡尔积）；专门运算（选择σ、投影π、连接（等值/自然连接））
3. 规范化理论：函数依赖；范式（1NF：属性不可再分、2NF：消除部分依赖、3NF：消除传递依赖）；范式升级目的（减少冗余、避免更新异常）

### 4. 数据库设计方法和步骤：需求分析、概念设计、逻辑设计和物理设计的相关策略。
1. 数据库设计步骤：需求分析→概念设计→逻辑设计→物理设计→实施→运行维护
2. 需求分析：收集用户需求、数据流图分析；方法（访谈/问卷）
3. 概念设计：设计E-R模型；方法（自顶向下/自底向上）
4. 逻辑设计：E-R模型转关系模型、规范化优化
5. 物理设计：确定存储结构、存取路径、索引设计
6. 数据库实施（建库/写程序/数据录入）；运行维护（备份恢复/性能优化/故障处理）
